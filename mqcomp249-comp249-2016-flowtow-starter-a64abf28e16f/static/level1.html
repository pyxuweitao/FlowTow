For this level you must write a simple web application that generates two pages. You are given a set of requirements below, these are encoded in a set of tests (level1_functional.py in the starter kit) that your application must pass. The tests correspond to the functional requirements below.

Functional requirements

  • Welcome to FlowTow

    As a visitor to the site, when I load the home page I see the title text "Welcome to FlowTow".

  • Three Images

    As a visitor to the site, when I load the home page I see three images displayed, each labelled with a date, a user name and the number of 'Likes' for the image.

    Each image will be contained in a <div> with class 'flowtow', that div will contain the user nick, date and number of likes each inside an element of class user, date and likes respectively.

  • Like Buttons

    As a visitor to the site, when I load the home page I see a button below each image with the text "Like".

    The button is part of a form that submits a like request. The action of the form should be /like, the form should have a hidden field called filename containing the filename of the image being liked and the button should be the submit button in the from.

  • Link to About Page

    As a visitor to the site, when I load the home page I see a link to another page with the link text "About".

  • About Page

    As a visitor to the site, when I click on the link "About" I am taken to a page that contains the site manifesto, including the words "FlowTow is a new, exciting, photo sharing service like nothing you've seen before!"

  • CSS Stylesheet

    As a visitor to the site, I notice that all the pages on the site have the same design with the same colours and fonts used throughout.

    Each page will have a similar structure and will link to a common CSS stylesheet.

Your Task

To achieve these requirements you will need to build a web application that is able to serve the two HTML pages (Home and About) and the associated stylesheet and images.

This level is all about the page structure and delivering two of the pages on the site - the main page and the about page. You won't use a database at this stage, when you generate the images on the main page you can hard-code the HTML in the templates. Later you will replace this with code to generate the image list from the database.

Your web application will be quite simple, it must serve the two URLs (/ and /about) as well as any static resources (CSS stylesheets, images) needed by your design. Most of the work will be in designing the page templates for your site.

The techniques to do this are covered here:

  • Python Web Applications.
  • Generating HTML Pages looks at the Bottle page templating system.
  • Testing Python Programs covers running unit tests.
