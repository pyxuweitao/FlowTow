Functional requirements

Note that there are no automatic tests for this level. I've not yet found a way to test file upload from Selenium. To verify the functionality you can perform the following tests manually.

As for level three plus:

  • As a registered user, when I load the "My Images" page I see a form that has a file selection input and a button labelled "Upload Image".
    • The upload form will have the id uploadform
    • The action attribute of the form will be /upload
    • The file field in the form will have the name attribute imagefile.
  • As a registered user, when I load the "My Images" page and select a file and then click on the "Upload Image" button my image file is uploaded to the site. The page that is returned is the "My Images" page and the newly uploaded image is the first image on the page.
  • As an unregistered user (not logged in) if I try to upload an image by accessing the /upload URL directly, I get a redirect response back to the home page.

Your Task

To achieve these requirements you will need to implement a file upload form that stores a new image file in the images folder.
